# Excel Model Style Guide

Reference document for Claude to produce investment banking-quality Excel models. This guide ensures consistency across all financial models generated by Claude.

---

## Color Coding Standards

### Font Colors

- **Blue (RGB: 0, 0, 255)**: All hardcoded inputs — numbers that users will change for scenarios
- **Black (RGB: 0, 0, 0)**: All formulas and calculations referencing same sheet
- **Green (RGB: 0, 128, 0)**: Links pulling from other worksheets within same workbook
- **Red (RGB: 255, 0, 0)**: External links to other files (rarely used)

### Cell Fills

- **Yellow/Light Orange (RGB: 255, 252, 199 or similar)**: Input cells requiring user attention
- **Dark Blue (RGB: 31, 78, 121)**: Section headers with white bold font
- **Light Gray (RGB: 217, 217, 217)**: Sensitivity table headers
- **Green (RGB: 146, 208, 80)**: Highlighted base case in sensitivity tables

---

## Border Hierarchy

### Section Separation

- **Thick bottom border**: End of major sections (e.g., after Sources and Uses, after Net Income)
- **Thin bottom border**: Subtotals and intermediate calculations within sections
- **No border**: Standard rows within a section

### Table Structure

- Year headers should have thin bottom borders
- Final row of each major section gets thick bottom border
- Sensitivity tables use light gray fill for headers

## Number Formatting

| Data Type | Format | Example |
|-----------|--------|---------|
| Years | Text or center-aligned | Year 1, Year 2 |
| Currency ($mm) | $#,##0 or #,##0 | $400 or 400 |
| Percentages | 0.0% | 25.0% |
| Multiples | 0.0x or 0.00x | 8.0x or 2.45x |
| Negative numbers | Parentheses | (50) not -50 |
| Dashes for N/A | em-dash centered | — |

## Section Headers

### Formatting

- Dark blue fill (RGB: 31, 78, 121)
- White bold font
- Left-aligned
- Merge across full model width
- ALL CAPS for header text

### Standard LBO Sections (in order)

1. TRANSACTION ASSUMPTIONS
2. OPERATING ASSUMPTIONS
3. DEBT SCHEDULE
4. INCOME STATEMENT
5. FREE CASH FLOW
6. RETURNS ANALYSIS
7. SENSITIVITY ANALYSIS

## Formula Construction Rules

### Core Principles

- Never hardcode values in formulas — all assumptions in dedicated input cells
- Use cell references, not numbers (e.g., =B5*(1+$B$6) not =B5*1.05)
- Interest calculated on beginning balance to avoid circular references
- Use MAX() and MIN() functions to prevent negative balances
- Use IFERROR() for division operations that could fail

### Consistency

- Same formula structure across all projection years
- Copy formulas horizontally, do not retype
- Lock row/column references with $ where appropriate

---

## Layout Standards

### Column Widths

- Column A (labels): 30-35 characters wide
- Column B (standalone inputs): 12-14 characters
- Year columns: 10-12 characters each

### Row Spacing

- 1 blank row between major sections
- No blank rows within sections unless for visual grouping
- Sub-headers (e.g., Senior Debt, Exit Assumptions) in bold, no fill

### Year Columns

- Year 0 = Entry/Close (no operating activity)
- Years 1-5 = Projection period
- Align year headers center

## Model Types and Required Sections

### LBO Model

- Transaction assumptions (entry multiple, leverage)
- Sources and Uses
- Operating projections (revenue, EBITDA, margins)
- Debt schedule with amortization
- Income statement through Net Income
- Free cash flow build
- Returns analysis (MOIC, IRR)
- Exit multiple sensitivity table

### DCF Model

- Operating assumptions
- Unlevered FCF projections
- Terminal value (Gordon Growth or Exit Multiple)
- WACC calculation
- Enterprise value bridge to equity value
- Sensitivity on WACC vs growth/exit multiple

### Operating Model / 3-Statement

- Revenue build (bottoms-up or top-down)
- Income statement
- Balance sheet
- Cash flow statement
- Working capital schedule
- Debt schedule

## Python Implementation Notes

When generating Excel files via openpyxl, use these style definitions:

```python
from openpyxl.styles import Font, PatternFill, Border, Side

blue_input = Font(color="0000FF")
yellow_fill = PatternFill("solid", fgColor="FFFFC7")
dark_fill = PatternFill("solid", fgColor="1F4E79")
white_bold = Font(bold=True, color="FFFFFF")

thick = Side(style='thick', color='000000')
thin = Side(style='thin', color='000000')
thick_border = Border(bottom=thick)
thin_border = Border(bottom=thin)
```

Always run `recalc.py` after generating to populate formula values and verify zero errors.
